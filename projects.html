Learning Journal PWA

This project is my Learning Journal Progressive Web App for FGCT6021 Mobile Application Development. The idea is to have a small web app where I can write weekly reflections, save them, and view them again later. I also experimented with different browser APIs to make it feel more interactive and “app-like”.

The app has three main pages: the home page, the journal page, and this projects page. The home page introduces the journal, the journal page handles writing and saving entries, and this page explains the main features and technical details of the project.

Main Features

  • Write new journal entries and tag them by week (Week 1–4 or Other).
  • Save entries so they stay even after refreshing or closing the browser.
  • Filter existing entries by tag, e.g. only show “Week 2” entries.
  • Word counter that updates as you type in the journal input box.
  • Copy the latest entry to the clipboard with one button.
  • Random learning quote from an external API to make the journal more motivating.
  • Light/Dark theme switch that remembers the user’s choice using LocalStorage.

APIs Used

Storage API

I used the LocalStorage API to store my journal entries and the current theme. Each entry is saved as an object containing the text, a tag, and the date. When the journal page loads, the app reads from LocalStorage and rebuilds the list of entries in the DOM. The theme selection (light or dark) is also saved so the site remembers it on the next visit.

Browser APIs

For browser APIs, I used the Notifications API and the Clipboard API. When an entry is saved, the app requests notification permission and, if allowed, shows a small message saying that the journal entry was saved. The clipboard feature lets me copy the most recent entry, so I can paste it somewhere else if needed.

Third-Party API

For the third-party API, I used the public Quotable API. When the user clicks the “Get a Quote” button on the journal page, the app sends a fetch request to the API and displays a random quote with the author’s name. This makes the journal page feel a bit more inspirational and dynamic.

How It Works (Technical Summary)

The project is built with HTML , CSS, and JavaScript. The header and footer are reused across all pages using JavaScript, and the theme is controlled by adding and removing CSS classes on the body element.

The journal logic is handled in a separate storage.js file, which manages saving, loading, and filtering entries. The browser-specific code (notifications and clipboard) lives in browser.js, and the external quote feature is in thirdparty.js. All of these are loaded on the journal page.

Overall, this project helped me practise DOM manipulation, event handling, and working with different types of web APIs inside a small PWA-style site.
